# J&J MedTech Sales Genie App

A Databricks-powered analytics stack for **Johnson & Johnson MedTech** surgical product sales. It loads curated sales datasets into Unity Catalog, exposes governed KPIs as metric views, and delivers a **natural-language “sales advisor”** via Databricks Genie plus a branded web app for reps and leadership.

## Business context

This demo models how a med-tech field organization reasons about **procedure volume**, **wallet share**, and **commercial opportunity** across accounts and surgeons:

- **HCP / surgeon lens** — Current-year and prior-year procedure volumes by specialty and product line, tied to NPI-level HCP records, so users can ask “who are the top surgeons by volume?” or compare specialties.
- **Account & territory lens** — Opportunity dollars, rolling 12-month sales, penetration trends, GPO alignment, and geographic hierarchy (area → region → territory) for prioritizing **where to sell** and **how deep** the relationship is.
- **Product & upgrade lens** — Upgrade paths, competitive positioning, and **targeting tiers** that classify opportunities:
  - **Tier 1 — Upgrade** — Expand share with existing customers (move to newer platforms).
  - **Tier 2 — Competitive** — Win from competitors.
  - **Tier 3 — Market expansion** — New procedures or greenfield opportunity.

The sample data spans **four product families** (Alpha, Beta, Gamma, Delta Series) and **five U.S. areas** (Northeast, Southeast, Central, Northwest, Southwest). Together, the app answers questions like total opportunity by product line, GPO concentration, penetration decline, and top accounts—using **governed SQL** generated by Genie from plain English.

## Architecture

```
Raw CSVs → Unity Catalog Volume → Delta Tables (3) → Metric Views (7)
                                                          |
                              +---------------------------+---------------------------+
                              v                           v                           v
                    Genie Space (NL → SQL)         FastAPI (/api/chat, /api/dashboard)
                    + tables + metric views        React UI (build) or index.html fallback
```

1. **Pipeline job** — Loads CSVs, adds UC metadata (PKs, comments), creates metric views, then creates or updates the Genie space from exported JSON.
2. **Genie space** — Natural-language queries against the three base tables and seven metric views, with instructions, certified SQL examples, and benchmarks.
3. **Databricks App** — FastAPI serves the Genie conversation API (polling for completions and optional query results), dashboard aggregates over `medtech.sales.*`, and static React assets when `frontend/build` exists.

## What’s included

### Data pipeline (four tasks)

| Task | Source | Purpose |
|------|--------|---------|
| `setup_and_load` | `01_setup_and_load.sql` | Create/load three Delta tables from CSVs on a UC volume |
| `add_uc_metadata` | `02_add_uc_metadata.sql` | Primary keys and column documentation |
| `add_business_semantics` | `03_add_business_semantics.sql` | Seven Unity Catalog **metric views** for shared KPI definitions |
| `create_genie_space` | `04_create_genie_space.py` | PATCH/POST Genie space from `valentina_genie.json` (REST) |

The Genie task uses the job’s **default notebook environment** (`environments` / `client` in `resources/valentina_job.yml`).

### Datasets (catalog `medtech`, schema `sales`)

| Table | Approx. rows | Grain / role |
|-------|----------------|----------------|
| `hcp_procedure_volume` | 150 | NPI / surgeon — CY/PY procedure volume by specialty and product line |
| `product_upgrades` | 400 | Product-level upgrade opportunities, territory and account |
| `account_targeting` | 300 | Account opportunity, penetration, GPO, tiers, rolling sales |

**Metric views** (examples): `total_procedure_volume`, `yoy_procedure_growth`, `total_upgrade_opportunity`, `rolling_12_sales_metric`, `account_penetration`, `market_exposure`, `total_max_market` — used by Genie for consistent metrics and by analysts who query UC directly.

**Join keys:** `rep_id` and `account` across all base tables; `territory` links `product_upgrades` and `account_targeting`.

### Genie space

- **Tables:** the three base tables above (see `valentina_genie.json` `data_sources.tables`).
- **Metric views:** all seven metric views (`data_sources.metric_views`).
- **Guidance:** domain instructions (tiers, product lines, geography, abbreviations CY/PY, GPO, IDN, NPI), table-selection rules, join patterns, and query conventions (e.g., TOP 10, dollar formatting).
- **Samples:** sample questions, certified SQL examples, and benchmarks for evaluation-style checks.

After the first **create**, note the printed space ID and align `GENIE_SPACE_ID` in `src/app/app.yaml` and `src/app/app.py` if it changes.

### Web application

- **Stack:** FastAPI (`app.py`), optional React production build under `frontend/build`, fallback `index.html`.
- **Databricks Apps** (`src/app/app.yaml`): **user OAuth** enabled; SQL warehouse and Genie space IDs set via `env`.
- **APIs:**
  - `POST /api/chat` — Starts or continues a Genie conversation; polls until the message completes; returns reply text, SQL, and tabular data when available.
  - `POST /api/feedback` — Thumbs up/down (or clear) on a message via Genie feedback API.
  - `GET /api/dashboard` — KPIs and chart series (opportunity by product line, target type, area, top accounts, procedure volume by specialty, top surgeons) from `account_targeting` and `hcp_procedure_volume` via the SQL warehouse.
  - `GET /api/health` — Liveness and configured Genie space id.
- **UI:** J&J red header, sidebar (e.g., Chat, Dashboard, Contact Center), suggested prompts, markdown for answers, SQL display where returned.

## Quick start

### Prerequisites

- [Databricks CLI](https://docs.databricks.com/dev-tools/cli/install.html) with a profile that can deploy to your workspace.
- Default target in `databricks.yml` uses profile `free-edition-rleach` and workspace path `/Workspace/Shared/medtech-genie-app` — adjust `targets.dev.workspace` for your environment.

### Deploy and run

```bash
databricks bundle validate
databricks bundle deploy --auto-approve

# Loads data, metadata, metric views, and creates/updates Genie
databricks bundle run medtech_pipeline

# Deploy/start the Databricks App (see bundle for job name)
databricks bundle run medtech_ask_genie
```

### Logs and teardown

```bash
# Example: replace app name/profile if yours differ
databricks apps logs medtech-sales-ask-genie -p free-edition-rleach

databricks bundle destroy --auto-approve
```
